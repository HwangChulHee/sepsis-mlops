### 결정 7·8 (공통) — 런타임 순서 자기모순: 마스크가 ffill **이후**에 생성됨 ❌ HOLD

- **자기모순**: 같은 문서가 결정 8 검토요청 `:133`에서 **"마스크 생성이 ffill 이전(원본 NaN 위치)에서 이뤄지는지(순서 누수 — review 지적)"**라고 *스스로* 요구한다. 즉 `:129`(순서)와 `:133`(요구)가 정면 충돌. v1이 명시 지적(`h1_review.md` v1, PASS 노트 결정7·8 + 새위험5)한 바로 그 항목이 **반영되며 거꾸로 적혔다** = 회귀.
- **근거**: `h1_decisions.md:129` vs `:133`; v1 검토(901c37f) 결정7 주의·새위험5; 마스크 의미는 `research/02:63`(GRU-D), `research/04` 치료행동 누수.
- **왜 HOLD인가**: 마스크는 H3 옵트인이지만 **H1 산출물 = "생성 코드 + off 플래그 자리"**다. 설계 스펙이 모순/오순서면 구현이 틀린 마스크 코드를 만들고, H3 검증(A·B)이 무의미해진다. 사소한 수정이나 **문서에서 먼저 고쳐야** 핸드오프 자립성이 선다.
- **제안**: 런타임 순서를 **`피처셋 슬라이스 → 분할 → (옵트인 시)마스크[원본 NaN에서] → (GRU)ffill → train평균 fill → train통계 정규화 → 윈도잉`**으로 정정(마스크를 ffill 앞으로). 결정 7·8 양쪽 문구를 일치시켜 모순 제거.
