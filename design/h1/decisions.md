## 결정 1: 피처 선택 — 활력 7 + 나이·성별 2 + 핵심 검사 9, 피처셋 2분기

- **결정**: 모델 입력 후보 = **활력 7**(HR, O2Sat, Temp, SBP, MAP, DBP, Resp) + 나이·성별 + **핵심 검사 9**(WBC, BUN, Platelets, Lactate, Creatinine, Glucose, PTT, HCO3, Calcium). 피처셋 2분기: **활력셋(9)** vs **활력+검사셋(18)**. ICULOS·Unit1·Unit2·HospAdmTime·**EtCO2** 제외.
- **근거 + 출처등급**:
  - 검사 9종 선정 근거 = **"임상 해석성 + 상대적 저결측 + 선행연구 지지"의 혼합**(단일 중요도 기준 아님) [우리 결정]. WBC·BUN·Platelets는 선행연구 지지 강함 [확인됨(review 989ee28 경유 1차대조)]; Lactate·Creatinine·Glucose는 임상 핵심이나 고결측이라 중요도 낮게 잡힘(Lactate 97.3% 결측) [확인됨: EDA]; PTT·HCO3·Calcium은 선행연구 고중요도라 추가 [확인됨(review 경유 1차대조)].
  - **EtCO2 제외**: 96.3% 결측 + 92% 미측정 → ffill로 끌어올 값조차 없는 *죽은 채널* [확인됨: EDA / review 권고].
  - ICULOS 제외 = 우측 절단 컨닝 차단 [확인됨: research/04 + EDA §6].
- **고려한 대안**: 검사 6종(v1) → 약한 3종으로 ablation 시 오결론 위험. 26종 전부 → 노이즈 폭증. 활력만 고정 → 검사 신호 못 봄.
- **미결/옵션**: 없음.
- **검토 요청 항목**: 9종이 캐시(결정 8)에 모두 존재하는지, 활력셋(9)이 활력+검사셋(18)의 컬럼 부분집합으로 정확히 파생되는지.

## 결정 2: 결측 처리 — 모델별 분기, 0-fill 금지

- **결정**: 트리(XGBoost·LightGBM)는 **NaN 그대로**. GRU는 **환자 내 forward-fill**(과거→미래만) → 잔여 빈칸은 **train split 평균**. **0으로 안 채움.**
- **근거 + 출처등급**: 트리 결측 내장 처리(XGBoost·LightGBM 모두) [ML 일반]; ffill 챌린지 관행 [확인됨: research/02 — Pou-Prom[3]·Fu[4], review 코드 대조]; 0 = 진짜 값(혈압 0=사망) [우리 결정/도메인].
- **고려한 대안**: 두 모델 동일 처리 → 한쪽 강점 죽음.
- **미결/옵션**: 없음.
- **검토 요청 항목**: ffill이 미래→과거로 새지 않는지, train 평균이 train split에서만 계산되는지.

## 결정 3: 정규화 — train-only z-score + 가벼운 클리핑(정규화 前)

- **결정**: 생리 범위 밖 극단값을 **가벼운 클리핑(정규화 이전)** → z-score 표준화, **평균·표준편차는 train split에서만**.
- **근거 + 출처등급**: train-only = 누수 방지 [ML 기본]; 범위 밖 값 0.1% 미만이라 가벼운 클리핑으로 충분 [확인됨: EDA §7]; 클리핑이 정규화 *앞*이어야 통계가 극단값에 안 휘둘림 [ML 일반].
- **고려한 대안**: 전체 데이터 정규화 → 누수. 공격적 이상치 제거 → 불필요(EDA).
- **미결/옵션**: 클리핑 경계 수치 = 핸드오프 디테일.
- **검토 요청 항목**: cross-site(A→B)에서 정규화 통계가 **A-train에서만** 잡히는지(결정 5·8 연동).

## 결정 4: 학습 단위 — many-to-many(전체 시퀀스, 매 시점 예측) + 절단 누수 진단·이연

- **결정**: 8시간 sliding window를 **폐기**하고, **환자 전체 시퀀스를 GRU에 흘리며 매 시각마다 예측**한다(many-to-many). 라벨 = **매 시점 SepsisLabel**(Method A, per-timestep). 가변 길이 배치는 **정렬용 패딩 + loss 마스킹**(패딩 위치는 손실 기여 0)으로 처리. **절단 누수는 H1에서 진단·명시만, 칼질(k드롭/예측구간 라벨)은 H3 측정 후 결정.**
- **근거 + 출처등급**:
  - **챌린지가 매 시간 간격마다 예측을 요구** — utility score도 매 시점 예측에 매겨짐 [확인됨: PhysioNet/CinC 2019 공식]. 즉 "매 시점 예측"은 우리 선택이 아니라 태스크 정의.
  - 완전 8윈도우는 **첫 양성 ICULOS<8 환자의 초기 양성 시점을 타깃에서 누락** — 그 강신호가 학습에 안 쓰임 [확인됨: EDA — 첫양성 p25=7(약 1/4이 ≤7), 입실즉시양성 370명(12.6%)]. many-to-many는 모든 시점이 타깃이라 **이 손실이 0**.
  - 전체 시퀀스를 한 번 흘려 모든 시점을 뽑으므로 매 시점 sliding보다 효율적(O(T) vs O(T×W)) [우리 결정]. *초기발병 보존은 sliding-many-to-one도 동일하므로, m2m의 **고유** 우위는 이 효율 하나다(정직한 한정 — review v3).*
  - ⚠️ **정정(커밋 전 검증)**: "상위팀 다수가 many-to-many RNN을 썼다"는 주장은 1차 근거가 없어 **철회**. 오히려 우승팀은 LightGBM(트리)+signature이고 상위권에 트리·윈도우요약 접근이 많음 [확인됨: 검색 1차대조]. many-to-many는 "유일한 정공법"이 아니라 위 세 근거(태스크 정합·초기발병 보존·효율)에 따른 선택.
  - 모든 환자 ≥8h라 시퀀스 자체는 항상 충분 [확인됨: EDA].
  - **정렬용 패딩 ≠ 윈도우 채움 패딩**: 후자는 패딩 마스크가 "기록 끝에 걸침"을 흘려 절단 누수 통로가 됨(그래서 기각). 전자는 단지 배치 길이 정렬용이고 **loss 마스킹으로 학습 0 기여** → 모델이 신호로 못 봄, 누수 아님 [우리 결정].
  - ⚠️ **패딩 무누수의 전제(자립형 명문화)**: 위 "패딩 누수 아님"은 **GRU가 단방향(causal: 시각 t 예측에 1..t만 사용) + 패딩이 우측(시퀀스 끝)** 일 때만 성립한다. 양방향이면 미래 정보 + 우측 패딩이 예측에 섞여 loss 마스킹으로도 못 막는다. **본 프로젝트 GRU는 단방향, 패딩은 우측으로 고정**한다 [우리 결정 — review v3 HOLD 반영]. (핸드오프는 자립형이므로 스모크가 단방향이라는 사실에 의존하지 않고 명시한다.)
  - **절단 누수 진단** [확인됨/우리 결정]:
    - *직접 경로*("곧 끝남" 컨닝): ICULOS 제거 + 마스크 OFF로 차단됨. many-to-many로 바꿔도 끝 정보가 새로 들어오지 않음.
    - *간접 경로*(측정밀도/ffill 신선도): 결정 7 마스크 OFF로 부분 차단, 우리 데이터 발현은 EDA로 확인.
    - *분포 편향*(양성이 발병 직전에 몰림): 라벨 정의상 자명, 누수 아닌 학습 한계. pos_weight로 완화(부속 결정).
- **고려한 대안**: (A) 완전 윈도우 → 초기발병 12~25% 손실. **(B) 매 시점 sliding-many-to-one(가용 lookback)** → 초기발병 손실은 동일하게 0이나 매 시점 윈도우 재계산으로 비효율 — *동등 대안이되 m2m이 효율 우위*. 윈도우 8→4 축소 → 손실은 줄지만 추세 맥락도 단축. 자투리 패딩 채움 → 절단 누수.
- **미결/옵션**: 없음(분기 해소).
- **검토 요청 항목**: 라벨이 per-timestep으로 정렬되는지, **loss 마스킹이 패딩 시점을 학습·평가에서 실제로 빼는지**, 가변 길이 처리(pack/pad)가 정확한지, 매우 긴 시퀀스(최대 331h)의 메모리·절단(truncated BPTT 등) 처리.
- **H1 진단 EDA 항목(핸드오프 포함)**:
  1. **측정밀도 누수 확인**: 양성 구간 vs 그 외 구간의 시간당 검사 측정 횟수(또는 결측률) 비교.
  2. **양성 시점 위치 분포**: 분포 편향 자명성 집계 확인.
  *(초기발병 비율은 이미 측정(p25=7, 12.6%)되어 many-to-many 채택 근거로 사용 → 별도 항목에서 제외.)*

## 결정 5: 분할 — 환자 단위, A→B 단방향 cross-site, 3분할

- **결정**: 분할은 **환자(파일) 단위**. cross-site는 **A→B 단방향**(B→A 폐기). 3분할: **A-train(학습) / A-val(튜닝: 임계값·조기종료) / B(평가, 끝까지 봉인)**. 통합 random 모드도 별도 유지(in-site 참고).
- **근거 + 출처등급**:
  - 환자 단위 = 누수 방지 [ML 기본].
  - 튜닝을 타깃 B에서 하면 타깃 누수 → 튜닝은 **A-val로만**, B는 평가 때만 [확인됨: review HOLD 3].
  - A/B 유병률 다름(A 8.8% vs B 5.71%)이라 cross-site가 일반화의 핵심 시험대 [확인됨: EDA §8 / research/01·04].
- **고려한 대안**: 행 단위 random → 환자 걸침(누수). 양방향 → 단순화 위해 단방향. val 출처 미정(v1) → 타깃 누수(HOLD 3).
- **미결/옵션**: A-train/A-val 비율·seed = 핸드오프 디테일.
- **검토 요청 항목**: **B가 학습·튜닝·정규화·pos_weight 어디에도 닿지 않는지**(평가 1회만), A-val이 A-train과 환자 겹치지 않는지.

## 결정 6: baseline — gradient boosted trees(XGBoost + LightGBM 병행), many-to-many와 평가 단위 정렬

- **결정**: baseline = **gradient boosted trees, XGBoost와 LightGBM을 둘 다 학습**. 입력(요약통계 피처)은 **공유**하고 학습만 분리. **A-val 기준으로 대표 부스터를 선정**해 GRU의 비교 기준선으로 삼고, 두 부스터의 cross-site 거동 차이는 **부록**으로 기록. GRU가 매 시점 예측하므로 **트리도 매 시점 단위로 정렬** — 각 시각 t에서 **"t까지 최대 8h lookback 요약통계"** 한 행. 초기 시점(ICULOS<8)은 **가용 범위만 요약**(버리지 않음 → GRU와 정렬). NaN은 그대로. 요약통계 = 마지막값·평균·최소·최대 + 추세 통계(delta·range·variance).
- **근거 + 출처등급**:
  - **우리 요약통계 baseline의 근거 = 챌린지에서 흔히 쓰인 "트리 + 슬라이딩윈도우 요약피처" 접근**(여러 참가팀) [확인됨: review가 상위팀 1차 논문 Separatrix·TASP·Sepyd로 확인]. *근거는 "우승팀 정렬"이 아님*(아래 정정 참조).
  - ⚠️ **우승팀(Morrill) 출처 정정**: 단순 요약통계가 아니라 **path signature**(시계열을 시간정보 보존 고정벡터로 변환) + **도메인 파생피처**(ShockIndex·BUN/CR·PartialSOFA) + **LightGBM**을 사용 [확인됨: CinC2019-014; path signature가 "요약통계의 시간동역학 손실을 극복하려" 도입됐다는 후속문헌]. 기존 "우승팀이 요약통계 병용" 문구는 1차 출처로 뒷받침되지 않아 **삭제**.
  - **서사**: 우승팀의 signature와 우리 GRU(many-to-many)는 **같은 목적(시간정보 보존)을 다른 수단으로** 공략한다 — baseline(요약통계, 시간동역학 손실) vs GRU(흐름 보존)의 비교가 곧 이 분야 핵심 질문의 재현이다 [우리 결정].
  - **둘 다 돌리는 이유**: 비용이 거의 0(트리는 CPU 분 단위, 피처 공유라 추가비용≈학습 1회), XGBoost=결측 내장 처리 표준·우리 02 서술 기준 / LightGBM=**우승 계열 정렬**(우승팀 LightGBM 사용은 [확인됨])·대용량(155만 행) 속도. **성능 우위 주장 아님** — "강한 기준선 확보 + 우승 계열 포함"이 목적 [우리 결정].
  - 매 시점 정렬 = GRU와 평가 단위를 일치시켜 cross-site·utility 비교를 공정하게 [우리 결정].
- **고려한 대안**: 단일 부스터 고정(서사·강건성 손해) / 시각별 행(추세 못 봄) / 평탄화 128열(차원 폭발).
- **미결/옵션**: 없음. **ShockIndex 등 도메인 파생피처는 baseline 직접 투입 금지, H2 ablation 항목으로 분리**(가벼운 하한선 원칙).
- **검토 요청 항목**:
  - **대표 부스터 선정이 A-val에서만 이뤄지는지**(B 미사용 — 결정 5 누수 규칙). B 점수로 고르면 타깃 누수.
  - **추세 통계는 `delta`(마지막−처음)·`range`(max−min)·`variance`로 특정** — "slope(기울기)"는 상위팀 verbatim 미확인이라 채택하지 않음(review v3). 이 3종은 추세를 표현하되 출처 의존이 없는 표준 통계 [우리 결정]. 전부 NaN인 lookback의 요약 처리(전부 NaN → NaN 유지).

## 결정 7: 결측 마스크 — 기본 OFF (옵트인 자리만)

- **결정**: 결측 마스크 채널 **기본 미사용**. H1은 생성 코드 + off 플래그 **자리만**. 켜는 검증(A·B)은 H3.
- **근거 + 출처등급**:
  - 측정 유무/빈도가 정보를 담는다(informative missingness)는 확립된 현상 [확인됨: Sharafoddini 2019, JMIR 2025].
  - 관찰-과정 피처가 **cross-site 이식성을 악화**(우리 A→B와 같은 우려) [유도: medRxiv 2026 MIMIC-IV↔eICU — 단 사망예측·cross-DB라 우리 발병/cross-site엔 직접 아님, 방향성만 유도].
  - 치료행동 누수 통로(Epic 외부검증 실패) [확인됨: research/04].
  - PhysioNet에서의 구체 발현은 결정 4 EDA로 확인 [검증 필요 → 데이터로].
- **고려한 대안**: 기본 ON → 미검증 누수 채택. 미구현 → H3 검증 불가.
- **미결/옵션**: 채택 여부는 H3 검증 결과에 달림.
- **검토 요청 항목**: off가 진짜 기본인지, 마스크 생성이 미래 정보를 안 쓰는지, **마스크가 ffill 이전(원본 NaN)에서 생성되는지**(결정 8 순서).

## 결정 8: 캐싱 레이어 — raw(NaN 보존) 한 번, 변환은 런타임 (순서 정정)

- **결정**: 후보 피처를 **NaN 그대로** + 라벨 + 사이트로 환자별 raw 배열로 **한 번** 캐싱(EtCO2 포함 전체 후보 보존). 4만 파일 읽기는 이때 한 번뿐. 피처셋 선택·분할·결측·정규화·시퀀스 구성은 **런타임**. **런타임 순서**:
  - 공통: 피처셋 슬라이스 → 분할
  - GRU 경로: **(옵트인 시) 마스크 생성 [원본 NaN 위치에서]** → ffill → train평균 fill → 클리핑 → train통계 정규화 → 시퀀스 구성(정렬 패딩)
  - 트리 경로(XGBoost·LightGBM): 매 시점 lookback 요약통계(NaN 그대로)
  - *핵심: 마스크는 ffill **이전**에 떠야 informative-missingness를 보존한다(v2 회귀 정정).*
- **근거 + 출처등급**: NaN 보존해야 두 경로 커버 [우리 결정]; 풀셋 캐싱 → 활력셋 슬라이스 파생 [우리 결정]; 재학습 다회 비용 절감 핵심 레버, CPU로 충분 [우리 결정].
- **고려한 대안**: ffill·정규화까지 캐싱 → 모델별 분기·분할별 정규화 불가(누수/경직). 마스크를 ffill 뒤에 생성(v2) → informative-missingness 훼손(review HOLD).
- **미결/옵션**: 캐시 포맷(npz/parquet) = 핸드오프 디테일.
- **검토 요청 항목**: 캐싱 경계가 정규화를 분할에 묶는지(train-only), 캐시에 미래 정보가 새지 않는지, **마스크가 ffill 앞에서 생성되는지 코드로 확인**.

## 부속 결정 (review 추가 누락 반영)

- **클래스 불균형**: 양성 시점 1.8%로 심한 불균형. **pos_weight를 per-timestep BCE에 적용**(many-to-many 손실), 비율은 **A-train에서 산출**. H1은 산출 입력(클래스 비율 집계)까지; 적용은 H2. **다운샘플링은 기각** — 시점 단위 다운샘플은 환자 시퀀스의 시간 연속성을 깨고, 환자 단위 다운샘플은 소수 양성 환자를 더 줄임 [우리 결정].
- **완결성 / 이상 라벨 필터**: 전원 ≥8h라 길이 필터는 사실상 무해소. 단 **양성 블록이 정의(연속, 기록 끝에서 ≤10h)와 어긋나는 이상 환자**, **초기 전부-NaN 시점("유령" 시점)**을 점검·로깅한다(제외는 신중히, 기본은 보존+마스킹). [우리 결정]
