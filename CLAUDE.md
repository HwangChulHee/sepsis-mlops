> 이 파일은 매 세션 자동으로 읽힌다. 상세 규칙은 [`design/WORKFLOW.md`](design/WORKFLOW.md) 참고.

## 개발 워크플로우 (반드시 따를 것)

설계는 **3단계**로 진행한다: **[1] 설계결정문서(DDD) → [2] 레드팀 검토(게이트) → [3] 구현 핸드오프.**

### ⚠️ 검토를 요청받으면 — 레드팀 모드로
설계 문서 검토를 맡으면 **동의가 아니라 반박이 목적**이다. 끄덕이지 말고 구멍을 찾아라:
- `[확인됨]` 항목 → **1차 출처를 직접 대조**해 진위 확인. 출처가 주장과 다르면 지적.
- `[우리 결정]` 항목 → 논리 비약·미검증 전제·반례 점검.
- **데이터 누수** 경로가 새로 생기는지 점검(분할·정규화·피처·라벨 정렬).
- 빠진 결정이 없는지, `reports/eda_findings.md` 측정치와 모순되지 않는지.
- 각 지적에 **출처/파일 줄번호**를 댄다. 통과(PASS)와 보류(HOLD)를 명확히 가른다.
- **보류가 하나라도 있으면 구현으로 넘어가지 않는다.**

## 누수 방지 대원칙 (전 단계 공통)
- **환자 단위 분할** — 같은 환자가 train/test에 걸치면 안 됨(파일 1개 = 환자 1명).
- **train-only 정규화** — 평균/표준편차는 학습 split에서만 계산.
- **0으로 채우지 않음** — 의료에서 0은 진짜 값(예: 혈압 0 = 사망). 결측은 ffill→train 평균.
- **ICULOS 제외** — 우측 절단 구조를 컨닝하는 시간 단서.
- **결측 마스크는 기본 OFF(옵트인)** — 치료행동 누수 통로(근거: `research/04_*`). 검증 통과 시에만.
- **결측 처리는 모델별 분기** — XGBoost는 NaN 그대로, GRU는 ffill + train 평균.

## 평가 기조
정확도 버림(불균형 1.8% 양성). **PR-AUC 1차 + 공식 utility score + cross-site(A→B)** 로 일반화 직접 측정.
