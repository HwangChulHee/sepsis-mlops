# Q3 — 평가: "잘했다"를 무엇으로 재나

> **출처 정책.** **[확인됨]** = 공식 챌린지 페이지/논문 직접 확인 · **[유도]** = 공식 파라미터로부터 계산(공식 채점 코드로 재확인 권장) · **[우리 결정]** = 우리 판단 · **[검증 필요]** = Claude Code 재확인.
> 이 문서는 **읽고 이해하는 것**이 목적이라 사람 말 + 비유로 썼다.

---

## 한 줄 요약

정확도는 불균형 때문에 거짓말을 해서 **안 쓴다.** AUROC는 "빨리 맞혔나"를 못 봐서 **보조**로만. 1차 기준은 드문 양성에 집중하는 **PR-AUC**, 정식 채점은 빨리·정확히·헛경보 없이를 한 숫자로 재는 **utility score**(공식). 그리고 진짜 승부는 **cross-site(A로 배워 B에서 시험)** — 7년째 미해결인 병원 간 일반화를 직접 잰다.

---

## 0. 왜 평가지표가 중요한가

평가지표가 틀리면 **엉뚱한 걸 잘하는 모델**이 1등이 된다. "무엇으로 채점하느냐"가 모델 선택만큼 중요하다.

---

## 1. 정확도(accuracy)는 왜 못 쓰나 [우리 결정 / 도메인 상식]

양성(패혈증 위험)이 전체 시간의 **1.8%**뿐이다 [확인됨: 우리 EDA]. 모델이 **"전부 안전"이라고만 찍어도 정확도 98%**가 나오는데, 정작 환자는 한 명도 못 잡았다. → 정확도는 이 데이터에서 완전히 거짓말을 한다. **버린다.**

---

## 2. AUROC — 순위는 보지만 타이밍·불균형에 약함

**AUROC가 보는 것**: "진짜 위험한 사람"의 위험 점수가 "안전한 사람"보다 높게 매겨지나. (위험/안전 한 명씩 뽑아 "누가 더 위험?" 물었을 때 맞히는 비율. 0.5=찍기, 1.0=완벽.)

**약점 두 가지**:
1. **다수에 후하다.** AUROC는 "안전한 다수를 잘 거르는 것"도 점수에 크게 넣는다. 우리 데이터는 안전이 98%라, 다수만 잘 맞혀도 점수가 높아 보인다. → **불균형에서 착시.**
2. **타이밍을 못 본다.** 발병 6시간 전에 맞힌 거나 발병 후에 맞힌 거나 똑같이 "맞음"으로 친다. 패혈증은 "빨리"가 핵심인데 그걸 무시한다.

→ **보조로만** 쓴다(논문들이 다 보고하니 비교용).

---

## 3. PR-AUC — 드문 양성에 집중 [1차 기준]

용어 둘:
- **잡아낸 비율(recall)**: 실제 위험한 사람 중 내가 몇 %를 잡았나.
- **경보 정확도(precision)**: 내가 위험하다고 경보한 것 중 진짜가 몇 %였나(나머지는 헛경보).

**PR-AUC**는 이 둘만 본다 — 안전한 다수를 거르는 건 **점수에 안 넣는다.** 오직 "드문 위험을 잘·깨끗하게 잡았나"에 집중. (비유: 건초더미에서 바늘 찾기. PR-AUC는 바늘만 보고 건초는 신경 안 쓴다.)

### 결정적 차이 — 멍청한 모델의 점수
| | 아무렇게나 찍는 모델 |
|---|---|
| AUROC 기준 | **0.5** (고정) |
| PR-AUC 기준 | **양성 비율 ≈ 0.018** |

같은 모델이 **AUROC 0.8 / PR-AUC 0.2**일 수 있다. AUROC는 후하게, **PR-AUC는 박하게(정직하게)** 준다. 불균형이 심할수록 차이가 벌어진다. → 그래서 PR-AUC를 1차 거울로 쓴다.

---

## 4. utility score — 임상 가치를 점수화 (이 문제의 공식 채점)

PR-AUC·AUROC가 못 보는 **"언제 맞혔나(빨리?)"**를 점수에 직접 넣은 지표. 챌린지가 이걸 정식 채점으로 썼다 [확인됨].

### 비유 — 일기예보 상금 게임
매 시간 "곧 폭풍 와요/안 와요"를 외치고, 외칠 때마다 상/벌을 받는다.
- 폭풍 **미리** 경고 → 큰 상금 · **닥쳐서** 경고 → 적은 상금 · **지나서** 경고 → 벌금 · 오는데 **안 온다고** 함 → 벌금
- 폭풍 없는데 "와요!" → 작은 벌금(자원 낭비) · "안 와요"(정답) → 0
("폭풍" = "패혈증")

### 네 가지 경우 [확인됨: 공식 페이지]
- **U_TP**: 패혈증 환자한테 "위험" 예측 → 상황 따라 상/벌
- **U_FP**: 정상 환자한테 "위험"(헛경보) → 작은 벌점
- **U_FN**: 패혈증 환자한테 "안전"(놓침) → 벌점
- **U_TN**: 정상 환자한테 "안전"(정답) → 0

### 점수 값 — 발병 시각 기준
**공식 파라미터** [확인됨]: 보상 창 = 발병 **12시간 전 ~ 3시간 후**, 최대 보상 **+1.0**, 너무 이른(12시간 전보다) 예측 벌점 **−0.05**, 늦은 예측(놓침) 최대 벌점 **−2.0**, 헛경보 벌점 **−0.05**.

점수는 **구간별 직선(선형 보간)**으로 정해진다. 아래 공식만 있으면 **어떤 시점이든** 직접 계산할 수 있고, 이어지는 표는 그 공식의 예시일 뿐이다 [유도 — Claude Code가 `evaluate_sepsis_score.py`로 재확인].

#### 원리 — 세 구간의 직선

`발병 N시간 전`을 기준으로(N>0이면 발병 전, 음수면 발병 후):

**"위험"으로 예측 (맞힘, U_TP)** — 두 직선:
- **상승 구간** (발병 12시간 전 → 6시간 전): `점수 = (12 − N) ÷ 6`
  - 12시간 전부터 한 시간마다 **+1/6 ≈ +0.167**씩, 6시간 전에 +1.0 도달.
- **하강 구간** (발병 6시간 전 → 3시간 후): `점수 = 1 − (6 − N) ÷ 9`
  - 6시간 전(+1.0)부터 한 시간마다 **−1/9 ≈ −0.111**씩, 3시간 후에 0.
- 12시간 전보다 이르면 −0.05, 3시간 후보다 늦으면 0.
  - 엄밀히는 상승 직선값을 −0.05에서 하한 클리핑(`max(점수, −0.05)`). 그래서 12시간 전 **정각은 0**, 그보다 이르면 점수가 −0.05까지 내려가 고정된다(시간 단위 데이터에선 13시간 전부터 −0.05).

**"안전"으로 예측 (놓침, U_FN)** — 한 직선:
- **벌점 구간** (발병 6시간 전 → 3시간 후): `점수 = −(6 − N) × 2 ÷ 9`
  - 6시간 전(0)부터 한 시간마다 **−2/9 ≈ −0.222**씩, 3시간 후에 −2.0.
- 6시간 전보다 이르면 0(아직 벌점 없음).

> 직관: **발병에 가까워질수록 맞혀도 보상이 줄고(−1/9), 놓치면 벌이 커진다(−2/9).** 놓침이 맞힘보다 2배 빠르게 나빠진다 → "빨리 맞혀라"를 점수가 계속 압박.

#### 예시 표 (위 공식으로 계산)

**패혈증 환자일 때**

| 예측 시점 (발병 기준) | "위험" 예측 (맞힘) | "안전" 예측 (놓침) |
|---|---:|---:|
| 12시간 전보다 이전 | −0.05 (너무 이름) | 0 |
| **12시간 전** | **0** (보상 창 열림) | 0 |
| 9시간 전 | +0.50 | 0 |
| **6시간 전 (최적)** | **+1.00** (최대 보상) | 0 |
| 5시간 전 | +0.89 | −0.22 |
| 4시간 전 | +0.78 | −0.44 |
| 3시간 전 | +0.67 | −0.67 |
| 2시간 전 | +0.56 | −0.89 |
| 1시간 전 | +0.44 | −1.11 |
| **발병 시점(0)** | +0.33 | −1.33 |
| 1시간 후 | +0.22 | −1.56 |
| 2시간 후 | +0.11 | −1.78 |
| **3시간 후** | **0** (보상 창 닫힘) | **−2.00** (최대 벌점) |
| 3시간 후보다 이후 | 0 | 0 (창 밖) |

> **왜 "6시간 전"이 최고점인가**: 라벨이 발병 6시간 전부터 1로 켜지게 shift돼 있어서, 그 시점이 "이상적으로 감지해야 할 때"다. 거기서 보상이 +1.0으로 최대.
> **놓침 벌점이 −2.0까지 큰 이유**: "늦게 맞히느니 차라리…"가 아니라, 발병이 임박했는데 계속 "안전"이라 하면 가장 위험하다는 임상 철학. 헛경보(−0.05)보다 **40배 무겁게** 벌한다.

**정상 환자일 때**

| 예측 | 점수 |
|---|---:|
| "위험"(헛경보) | −0.05 (항상) |
| "안전"(정답) | 0 |

### 점수의 척도 (정규화) [확인됨]
총점을 그대로 안 쓰고 0~1로 눈금을 맞춘다:
```
정규화점수 = (내 총점 − 무행동 총점) / (완벽 총점 − 무행동 총점)
```
- **1.0** = 완벽 · **0.0** = 아무것도 안 함(전부 "안전"만) · **음수** = **아무것도 안 하느니만 못함(해로움)**

이 척도가 강력한 이유: "0이 기준선"이라 직관적이다. 0보다 높으면 안 하는 것보단 낫고, **0보다 낮으면 있는 게 없는 것만 못하다.** 의료의 "해를 끼치지 마라" 원칙과 맞는다.

### 충격적 현실 [확인됨]
- 완벽 = 1.0인데 **챌린지 우승 ≈ 0.36** (완벽의 1/3)
- **처음 보는 병원(C)에선 음수** — 우승 모델조차 새 병원에선 "입 다무는 게 나았다"
- 7년간 Transformer까지 동원해도 이 천장이 안 올라간다 → 진짜 난관은 모델이 아니라 **일반화**

### 우리 고민 하나를 자동으로 풀어줌
앞서 "발병 후 데이터를 자를까?" 고민하다 "자르면 t_sepsis(미래 정보) 누수니 자르지 말고 평가로 처리하자"고 결론 냈다. utility가 **늦은 예측에 −2.0 벌점**을 줘서 발병 후 맞힘을 자동으로 깎으므로, 데이터를 손대지 않아도 지표가 알아서 처리한다. **우리 설계 결정과 공식 지표가 일치.**

## 5. cross-site — 병원 간 일반화 (Q3의 하이라이트)

공식 논문이 보여준 것 [확인됨]:
- utility 점수가 학습데이터를 준 **두 병원(A·B) 사이엔 강한 상관(ρ=0.949)**.
- 학습데이터를 안 준 **세 번째 병원 C와는 거의 무상관(ρ≈−0.03, 0.01)**.

쉽게: **A에서 잘하면 B에서도 잘한다. 그런데 A·B에서 잘하는 게 C에서 잘하는 것과는 아무 상관이 없다.** 처음 보는 병원에선 성능이 따로 논다.

**우리 차별점**: 챌린지엔 C가 비공개였지만 우리 손엔 A·B가 있다. **A로 학습 → B로 평가**해 이 "다른 병원 일반화"를 직접 측정한다. 2019 팀들이 점수만 내고 끝낸 그 격차를 우리는 처음부터 숫자로 본다. (배포 후 다른 환경에서 무너지나 = 운영 관점과 직결.)

> ✅ [확인됨: 원논문 직접 대조] 논문엔 상관 분석이 둘 있다 — ①AUROC↔utility(사이트 내) ②사이트 간 utility 전이. 위 ρ=0.949/−0.033/0.013은 ②번. 이전 논의에서 댄 ρ=0.791/0.839(A·B), 0.054(C)는 ①번 수치. 둘 다 논문에서 확인 완료.

## 6. 우리 평가 방침 [우리 결정]

| 지표 | 역할 | 왜 |
|---|---|---|
| 정확도 | **안 씀** | 1.8% 불균형에서 거짓말 |
| AUROC | 보조 | 타이밍·불균형 못 봄, 비교용 |
| **PR-AUC** | 1차 기준 | 드문 양성 집중, 빠름, 정직 |
| **utility score** | 정식 채점 | 빨리·정확·헛경보 다 반영, 논문 비교, 발병 후 자동 처리. 공식 코드 사용 |
| **cross-site (A→B)** | 일반화 측정 | 7년째 난제 직접 정량화 (차별점) |

## 7. 면접 한 줄

> "정확도는 불균형 때문에 못 쓰고, AUROC는 조기성을 못 봐서, 공식 utility를 정식 지표로 썼습니다. 1이 완벽, 0이 '아무것도 안 함', 음수가 '해로움'이라 해석이 명확합니다. 우승팀도 0.36, 새 병원에선 음수였다는 게 이 문제가 모델이 아니라 일반화의 문제임을 보여줍니다. 그래서 A로 학습해 B로 평가하는 cross-site 검증을 처음부터 넣었습니다."
