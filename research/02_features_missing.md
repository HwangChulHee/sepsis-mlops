# Q2 — 피처와 결측: 무엇을 먹이고, 빈칸을 어떻게 채울까

> 이 문서는 **읽고 이해하는 것**이 목적이라 사람 말로 썼다.
> 신뢰도를 알 수 있게 각 결정에 출처 등급을 달았다:
> **[확인됨]** = 공식 논문/페이지를 직접 확인 · **[우리 결정]** = 근거 위에서 우리가 판단(논문 인용 아님) · **[검증 필요]** = Claude Code가 1차 출처로 재확인할 것.

---

## 한 줄 요약

믿을 만한 활력징후 + 나이·성별을 기본으로 쓰고, 피검사는 핵심만 골라 얹는다.
빈칸은 **모델마다 다르게** 다룬다 — XGBoost는 알아서 처리하니 그냥 주고, GRU는 직전 값으로 채우되 "여긴 비었었다"를 따로 표시한다. **0으로는 절대 안 채운다.**
이 모든 결정의 목적은 하나 — **"시험은 잘 보는데 실전에선 무너지는" 가짜 성능을 막는 것.**

---

## 0. Q2가 답하는 질문

딱 두 가지다. **(1) 모델한테 뭘 먹일까(피처 선택), (2) 빈칸을 어떻게 채울까(결측 처리).**
데이터엔 측정값이 40종류 있고 [확인됨: 논문 — 활력 8+검사 26+인구 6], 측정 컬럼(34개) 기준 약 80%가 빈칸이다 [확인됨: 우리 EDA로 직접 측정 = 80.1%]. 그래서 이 두 질문이 중요하다.

---

## 1. 무엇을 먹일까 (피처 선택)

측정값 40종류는 세 덩어리로 나뉜다.

- **자주 재는 것 — 활력징후 8개** (심박·혈압·체온·호흡 등). 기계가 계속 모니터링해서 빈칸이 적다 [확인됨: 활력징후는 시간당 갱신].
- **가끔 재는 것 — 피검사 26개** (백혈구·젖산 등). 피 뽑아 검사실 보내야 해서 하루 한 번 잴까 말까, **90% 넘게 빈칸** [확인됨: 검사수치는 하루 단위 갱신].
- **거의 안 변하는 것 — 나이·성별 등**.

### 결정: 활력징후 + 나이·성별을 기본, 피검사는 핵심만 [우리 결정]

**왜?** 피검사는 대부분 비어서, 쓰려면 "없는 값을 추측해서 채워야" 한다. 추측이 많아질수록 모델은 진짜 데이터가 아니라 우리가 지어낸 값으로 배운다. 활력징후는 빈칸이 적어 추측이 덜 들어가니 더 믿을 만하다.
**안 그러면?** 피검사 26개를 다 욱여넣으면 입력의 대부분이 "추측한 값"이 되어 노이즈만 늘고 진짜 신호가 묻힌다.

> ⚠️ 단, 이건 "이게 최선"이라는 검증된 출처가 아니라 **합리적 판단**이다. 그래서 가정으로 두지 않고 **실험으로 검증**한다 — "활력징후만" vs "핵심 피검사 추가"를 비교해서 우리 데이터로 직접 증명한다. (남이 했으니까보다 내 데이터로 해보니까가 더 강하다.)

### 결정: "몇 시간째인지(ICULOS)"는 뺀다 [우리 결정]

**왜?** 패혈증 환자는 기록이 짧게 끝나고 정상 환자는 길게 이어진다. "몇 시간째인지"를 주면 모델이 증상이 아니라 **"기록이 곧 끝나나?"를 그 숫자로 컨닝**한다. 실전에선 미래라 알 수 없는 정보다.
**안 그러면?** 시험 점수만 좋고 실제 병원에선 무너지는 가짜 성능이 나온다. 의심스러운 단서는 선제 차단한다.

---

## 2. 빈칸을 어떻게 채울까 — 모델마다 다르다

여기 핵심 깨달음이 있다: **모델마다 빈칸 다루는 방식이 다르다. 같게 하는 게 오히려 틀리다.**

- **XGBoost (베이스라인)**: 빈칸을 **스스로 처리하는 능력이 내장**돼 있다 [ML 일반: XGBoost의 알려진 성질]. 그래서 아무것도 안 채우고 **빈칸 그대로** 줘도 된다. 오히려 미리 채우면 그 내장 능력을 방해한다.
- **GRU (메인 모델)**: 빈칸(NaN)을 **아예 못 먹는다** [ML 일반: 신경망 기본 동작]. 그래서 채워줘야 한다:
  1. **직전 값으로 이어 채우기** (3시에 잰 값을 4·5시에도 끌고 감 — "forward-fill") [검증 필요: "이게 sepsis 연구 표준"이라는 주장은 Claude Code가 참가팀 논문으로 확인].
  2. 그래도 빈 곳(한 번도 안 잰 검사)은 평균값으로.
  3. "여긴 비었었다"를 따로 표시 (다음 항목).

**왜 분기하나?** 두 모델 성질이 실제로 다르기 때문. 한 방식으로 통일하면 XGBoost는 강점을 못 살리고 GRU는 망가진다. → **풀 학습에서 이 분기를 반드시 명시.**

---

## 3. "빈칸이었다"는 것도 정보다 (결측 마스크)

Q2에서 제일 재밌는 부분이다.

**의사가 어떤 검사를 자주 한다 = 그 환자가 의심스럽다**는 뜻이다. 멀쩡한 환자한텐 젖산 검사를 안 하지만, 패혈증이 의심되면 자주 잰다. 그러니 **"언제 측정했나/안 했나" 자체가 환자 상태를 알려주는 신호**다.

그래서 빈칸을 그냥 채워 없애면 그 신호도 사라진다. 대신 옆에 **"여긴 진짜 잰 값 / 여긴 비어서 채운 값"을 1·0으로 표시한 지도**(결측 마스크)를 같이 준다.

> 비유: 시험 답안지에서 "확실히 아는 답 / 찍은 답"을 따로 표시해주는 것과 같다. 채점자(모델)가 그 차이를 알면 더 똑똑하게 판단한다.

**[검증 필요]** 마스크가 항상 도움되는 건 아니다(과제·환자군에 따라 다름). 그래서 **마스크 있는 버전 vs 없는 버전을 비교**한다. 효과 관련 1차 출처는 Claude Code가 확인.

---

## 4. 빈칸을 0으로 채우면 안 되는 이유 [우리 결정 / 도메인 상식]

흔한 실수: "빈칸이니 0 넣자." **안 된다.**

의료에서 **0은 "측정 안 함"이 아니라 진짜 값**이다. 혈압 0 = 사망. 체온 0 = 불가능. 빈칸을 0으로 채우면 모델이 "이 환자 혈압 0이네 = 죽었네"로 오해한다.
**안 그러면?** 데이터를 통째로 잘못 읽는다. 가장 흔하고 치명적인 실수라 못 박는다. (그래서 0 말고 평균/직전 값으로 채우고, "비었음"은 마스크로 따로 표시한다.)

---

## 5. 크기 맞추기 & "학습 데이터에서만" [우리 결정 / ML 기본 원칙]

- **크기 맞추기(정규화)**: 심박은 60~100, 체온은 36~38… 범위가 제각각이면 모델이 큰 숫자에 휘둘린다. 그래서 비슷한 범위로 맞춘다.
- **단, 그 "평균·범위"를 학습 데이터에서만 계산**한다. 전체 데이터(시험 포함)에서 계산하면 시험 정보가 학습에 새어든다(누수) — 시험 전에 시험지를 살짝 본 셈.
- **이상한 값**: 심박 280 같은 값이 있지만 EDA상 0.1%도 안 돼서 [확인됨: out-of-bounds 극소수] 가볍게 잘라주면 충분.

---

## 6. 이 모든 걸 꿰뚫는 하나의 원칙

위 결정들은 따로 노는 게 아니라 **하나의 목적**을 향한다:

> **"시험은 잘 보는데 실전에선 무너지는" 가짜 성능을 막는다.**

- 추측 적은 피처만 → 노이즈성 가짜 성능 방지
- ICULOS 제외 → 컨닝성 가짜 성능 방지
- 0으로 안 채움 → 데이터 오해 방지
- 학습 데이터로만 평균 계산 → 누수성 가짜 성능 방지

이건 프로젝트 전체 주제(**배포해도 안 무너지는 신뢰할 수 있는 시스템**)와 같다. 정직한 데이터 처리 → 정직한 성능 → 실전에서 버티는 시스템.

**면접용 한 줄**: "전처리 결정들은 전부 '실전에서 무너지지 않게, 가짜 성능을 만드는 지름길을 하나씩 차단한다'는 한 원칙에서 나왔습니다."

## 7. 출처 정직성 — 뭐가 확실하고 뭐가 우리 판단인가

이 문서를 읽을 때 알아둘 것. (임의로 지어낸 건 없지만, 근거의 무게가 다르다.)

- **공식 출처로 확실한 것 [1]**: 활력징후 시간당·검사 일단위(논문 원문), 결측 의도적 보존(NaN), ICULOS 의미(데이터 문서), 그리고 **제3 병원(C)에서 AUROC와 임상가치(utility)가 무상관**(Spearman ρ=0.054; 단 같은 병원 A·B에선 ρ=0.79/0.84로 강한 상관 — 즉 "병원 간 일반적 무상관"이 아니라 **외부 병원에서만** 무너짐).
- **우리 EDA로 측정한 것(논문 수치 아님)**: 측정 컬럼 기준 결측 ~80%(80.1%), 공개 training set(A+B) 행 수 1,552,210 — 둘 다 우리가 직접 센 값이지 논문이 보고한 숫자가 아니다.
- **ML 일반 상식**: XGBoost 결측 내장 처리, 신경망이 NaN 못 먹음, 학습데이터로만 정규화, 의료에서 0 위험 — 분야 표준(교과서/알고리즘 원논문 수준).
- **우리 판단(아직 검증 전)**: 피처를 "활력징후+핵심 피검사"로 좁히는 것, 그 핵심 피검사 목록(WBC·Lactate·Creatinine·BUN·Platelets·Glucose), 마스크 채택 — **합리적이지만 "이게 최선"이라는 출처는 아직 없다.** → 피처는 실험으로, 기법은 Claude Code 출처 확인으로 메운다.

## 8. Claude Code 재검증 목록

- [x] forward-fill이 sepsis 연구의 흔한 관행 → **확인**: CinC2019-034(Pou-Prom, LOCF)·CinC2019-317(Fu, carry-forward) 등 다수 팀 사용. ("표준"보단 "흔한 선택지" — Zabihi 등은 선형보간 사용) [3][4]
- [x] 결측 마스크의 성능 효과 → **확인**: GRU-D(Che et al., *Sci Rep* 2018)가 mask+decay로 효과 입증. 단 Singh et al.(*JMIR Med Inform* 2021)은 sepsis 전향 세팅에선 효과 사라질 수 있음을 보고 → "도움되나 과제·환자군 의존"이 정확 [5][6]
- [x] 완결성 필터(거의 빈 윈도우 제외) 관행 → **확인**: Ding et al.(WWW '23, arXiv:2302.05702)이 결측 80%↑ 환자 제외 [7]
- [ ] 핵심 피검사 목록이 선행연구 피처 중요도와 일치하는지 교차 확인 (미완 — 다음 단계)
- [x] 행 수 "불일치" → **해소**: 논문은 1,424,171을 보고하지 않음(출처 불명·오기). 논문 수치는 "over 2.5M hourly windows"(A+B+C 전체 65,155명, 숨은 셋 포함). 우리 1,552,210은 공개 set A+B(40,336명)의 정확한 행 수 → **불일치 없음, 우리 카운트가 맞음** [1]
- [ ] (실험) "활력징후만" vs "핵심 피검사 추가" 성능 비교
- [ ] (실험) 마스크 유무 비교
