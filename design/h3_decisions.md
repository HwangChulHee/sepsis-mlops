# H3 설계결정문서 (DDD) — cross-site 평가 (B 개봉)

> **설계 근거**: H2 산출물(6조합 학습 모델·전처리통계·τ, MLflow) 위에서 **B를 처음 펼쳐** cross-site 채점. H2 결정은 [`h2_decisions.md`](h2_decisions.md)(완료), 결과는 [`reports/h2_results.md`].
> **워크플로우·출처등급**: [`WORKFLOW.md`](WORKFLOW.md). 검토(`h3_review.md`) 통과 후 핸드오프로.
> **상태**: 초안 — 레드팀 검토 전.

## 한 줄 요약

H2에서 A로 학습한 **6조합을 B(봉인 해제)로 채점**해 cross-site 일반화를 측정한다. **A 전처리통계·τ를 그대로 적용**(B 재튜닝 절대 금지), **B는 채점에만 1회 개봉**. 추가로 utility 구현을 공식 스크립트와 최종 대조하고, 마스크 누수 가설을 검증한다. **운영(H4)이 최우선**이므로 H3는 그 전 검증 단계.

## H3 범위 / 범위 외

| 범위 (H3가 하는 것) | 범위 외 |
|---|---|
| 6조합 A→B 채점(PR-AUC+utility), A-val→B gap | 양방향 B→A (보류) |
| utility 공식 스크립트 최종 대조 + 우승점수 분해 재현 | 서빙·드리프트·재학습 (H4, 최우선) |
| 마스크 누수 검증(마스크 ON 재학습→B 비교) | 피처셋 최종 확정 (H4 평가 후) |
| ⏸ gap·순위역전 해석(사람) | |

---

## 결정 1: 채점 대상 — 6조합 전부 A→B

- **결정**: H2의 **6조합(XGB·LGBM·GRU × vitals·vitals_labs)을 모두** B로 채점. 추론은 빠르므로 골라낼 이유 없음. PR-AUC + utility, **A-val→B gap**을 한 표로.
- **근거 + 출처등급**: 6조합 다 봐야 모델별·피처셋별 gap·순위역전을 완전 비교 [우리 결정]. H2 결과는 in-site라 본 시험대는 cross-site [확인됨: reports/h2_results.md, GRU A-val 0.40은 도메인 시프트 없음].
- **고려한 대안**: 대표만(GRU·XGBoost) 채점 → 피처셋·LGBM 비교 누락.
- **검토 요청 항목**: 6조합 모델·전처리통계·τ가 MLflow에서 정확히 로드되는지.

---

## 결정 2: 누수 규칙 — A 전처리·τ 그대로, B 재튜닝 금지, 1회 개봉 (★H3 핵심)

- **결정**: B 채점은 **A에서 산출·동결한 것만** 사용 — 정규화 μ/σ, fill 평균, 클리핑 범위, **τ** 전부 A(H2 저장 아티팩트). **B로 어떤 것도 재튜닝·재선정 금지**(τ 재탐색·정규화 재계산 모두 금지). **B는 채점에만 사용**, 모델 선택·하이퍼파라미터에 영향 줄 수 없음.
- **근거 + 출처등급**:
  - 실전(처음 보는 환자)은 과거(A) 기준으로 처리 — B 통계로 정규화하면 누수 [확인됨: ML 일반 / H1·H2 누수 원칙].
  - B를 한 번 보고 재튜닝하면 "시험지→연습지" 타락, cross-site 자기기만 [우리 결정 — 프로젝트 핵심 철학].
- **고려한 대안**: B에서 τ 재선정(점수는 오르나 누수 — 기각). B 통계로 정규화(누수 — 기각).
- **검토 요청 항목**: B가 전처리통계 산출·τ선정·모델선택에 닿는 경로가 정적·동적으로 차단되는지(동적 B-guard 역방향 — 이제 B가 *입력*이라 "B는 채점 함수에만, 학습/선정 함수엔 미진입").

## 결정 3: 평가 지표·gap — H2와 동일 잣대

- **결정**: PR-AUC(GRU masked) + utility, H2와 **동일 구현**. 보고는 **A-val → B 나란히 + gap = A_val − B**. 순위도 A/B 각각.
- **근거 + 출처등급**: 동일 잣대라야 gap이 의미 [우리 결정]. gap·순위역전이 핵심 발견(복잡한 GRU가 더 무너지나) [확인됨: cross-site 난제 — research/01].
- **고려한 대안**: B에서 새 지표 도입(비교 불가).
- **검토 요청 항목**: A-val 점수를 reports/h2_results.md에서 그대로 가져와 일관 비교하는지.

---

## 결정 4: utility 정밀 검증 — 공식 스크립트 최종 대조

- **결정**: B 채점 **전에** 우리 `eval/utility.py`를 **공식 `evaluate_sepsis_score.py`와 실제 데이터로 끝단 대조**(H2-a는 14행 표 검증, 여기선 공식 코드 자체와 동등성). + 우승팀 ~0.36 분해를 우리 구현으로 재현 시도(가능 범위).
- **근거 + 출처등급**: 채점 도구가 틀리면 B 점수 전부 거짓 → B 개봉 전 최종 확정 [우리 결정]. utility 정의는 이미 [확인됨: H2-a, 14행 표 일치].
- **고려한 대안**: H2-a 검증으로 충분하다 보고 생략 → 공식 스크립트 자체와 미대조 위험.
- **미결/옵션**: 공식 스크립트 입수 경로(PhysioNet 공개). 우승점수 분해는 데이터 가용 시.
- **검토 요청 항목**: 공식 스크립트와 동등성 판정 기준(동일 입력 → 동일 출력 ±tol).

---

## 결정 5: 마스크 누수 검증 — 마스크 ON 재학습 후 B 비교

- **결정**: 마스크 OFF(기본)와 **마스크 ON**을 비교. 마스크는 GRU 입력 채널이므로 **마스크 ON으로 재학습 필요**(단순 채점 아님). 두 버전을 **B로 채점**해 마스크가 cross-site 점수에 미치는 영향 측정. 마스크 ON이 B에서 크게 오르면 누수 의심(경계), 영향 미미하면 OFF 결정 확정.
- **근거 + 출처등급**: H1-c에서 측정밀도 누수 약함 확인(any-lab 1.05×, t0 직전 급증 없음) → cross-site에서도 확인 [확인됨: reports/h1_diagnostics.md]. 마스크는 default-off/opt-in 정책 [확인됨: H1 결정 7].
- **고려한 대안**: 마스크 검증 생략(H4 최우선이면 이연 가능) — 단 마스크 정책 근거가 in-site에만 머묾.
- **미결/옵션**: 대표 조합(예: GRU vitals)만 마스크 검증할지, 범위는 핸드오프. **H4 최우선이면 H3-c는 선택적**(b까지가 핵심).
- **검토 요청 항목**: 마스크 ON 재학습이 H2 학습설정과 동일 조건(τ·전처리 제외 동일)인지.

---

## 부속 결정

- **양방향(B→A) 보류**: A→B가 핵심 질문, B→A는 보강. **운영(H4) 최우선**이라 H3에선 A→B만 [우리 결정].
- **피처셋 최종 확정 이연**: H2에서 미결로 둔 메인 피처셋은 H3 cross-site 결과 + H4 운영 평가 후 결정. H3-b의 B gap이 그 핵심 재료.
- **사람 체크포인트(H3-b 후)** ⏸: A-val→B gap 표·순위역전을 보고 사람이 해석(GRU vs 트리 일반화, 피처셋 방향). 자동 진행 아님.

## H3 PASS 기준 (핸드오프에 박을 게이트 — 초안)

1. 6조합 B 채점 무오류 완주, PR-AUC+utility 기록.
2. **누수 가드**: B가 전처리통계·τ·모델선택에 미접촉(정적·동적), A 동결 아티팩트만 사용.
3. utility 공식 스크립트 동등성 통과(동일 입력→동일 출력 ±tol).
4. A-val→B gap 표 생성(H2 결과와 일관).
5. (H3-c 수행 시) 마스크 ON/OFF B 비교 수치 산출.
