## 결정 1: 피처 선택 — 활력 7 + 나이·성별 2 + 핵심 검사 9, 피처셋 2분기

- **결정**: 모델 입력 후보를 정한다. **활력 7**(HR, O2Sat, Temp, SBP, MAP, DBP, Resp) + 나이·성별 + **핵심 검사 9**(WBC, BUN, Platelets, Lactate, Creatinine, Glucose, PTT, HCO3, Calcium). 피처셋 2분기: **활력셋(9)** vs **활력+검사셋(18)**. ICULOS·Unit1·Unit2·HospAdmTime·**EtCO2** 제외.
- **근거 + 출처등급**:
  - 검사 9종 선정 근거를 **"임상 해석성 + 상대적 저결측 + 선행연구 지지"의 혼합**으로 명시(피처 중요도 단일 기준 아님) [우리 결정]. WBC·BUN·Platelets는 선행연구 지지 강함 [확인됨: review 901c37f가 1차 출처 대조]; Lactate·Creatinine·Glucose는 임상 핵심 지표지만 고결측이라 중요도가 낮게 잡힘(특히 Lactate 97.3% 결측) [확인됨: EDA]; PTT·HCO3·Calcium은 선행연구 고중요도라 추가 [확인됨: review].
  - **EtCO2 제외**: 96.3% 결측 + 92%가 한 번도 미측정 → ffill로 끌어올 값조차 없는 *죽은 채널*. GRU엔 사실상 상수, XGBoost엔 무해무익 [확인됨: EDA / review 권고].
  - ICULOS 제외 = 우측 절단 컨닝 차단 [확인됨: research/04 + EDA §6].
- **고려한 대안**: (a) 검사 6종(v1) → 약한 3종으로 ablation 시 "검사 무용" 오결론 위험(review HOLD 1). (b) 검사 26종 전부 → 노이즈·추측 폭증. (c) 활력만 고정 → 검사 신호 영영 못 봄. 그래서 9종 후보로 넓히고 ablation으로 비교(H2).
- **미결/옵션**: 없음(EtCO2·검사 구성 모두 해소).
- **검토 요청 항목**: 9종이 캐시(결정 8)에 모두 존재하는지, 활력셋(9)이 활력+검사셋(18)의 컬럼 부분집합으로 정확히 파생되는지.

## 결정 2: 결측 처리 — 모델별 분기, 0-fill 금지

- **결정**: XGBoost는 **NaN 그대로**. GRU는 **환자 내 forward-fill**(과거→미래만) → 잔여 빈칸은 **train split 평균**. **0으로 안 채움.**
- **근거 + 출처등급**: XGBoost 결측 내장 처리 [ML 일반]; ffill 챌린지 관행 [확인됨: research/02 — Pou-Prom[3]·Fu[4], review 코드 대조]; 0 = 진짜 값(혈압 0=사망) [우리 결정/도메인].
- **고려한 대안**: 두 모델 동일 처리 → 한쪽 강점 죽음.
- **미결/옵션**: 없음.
- **검토 요청 항목**: ffill이 미래→과거로 새지 않는지, train 평균이 train split에서만 계산되는지.

## 결정 3: 정규화 — train-only z-score + 가벼운 클리핑

- **결정**: z-score 표준화, **평균·표준편차는 train split에서만**. 생리 범위 밖 극단값 가벼운 클리핑.
- **근거 + 출처등급**: train-only = 누수 방지 [ML 기본]; 범위 밖 값 0.1% 미만이라 가벼운 클리핑으로 충분 [확인됨: EDA §7].
- **고려한 대안**: 전체 데이터 정규화 → 누수. 공격적 이상치 제거 → 불필요(EDA).
- **미결/옵션**: 클리핑 경계 수치 = 핸드오프 디테일.
- **검토 요청 항목**: **cross-site(A→B) 모드에서 정규화 통계가 train 사이트(A-train)에서만** 잡히는지(결정 5·8과 맞물림).

## 결정 5: 분할 — 환자 단위, A→B 단방향 cross-site, 3분할

- **결정**: 분할은 **환자(파일) 단위**. cross-site는 **A→B 단방향**으로 확정(B→A 폐기). 3분할: **A-train(학습) / A-val(튜닝: 임계값·조기종료) / B(평가, 끝까지 봉인)**. 통합 random 모드도 별도로 유지(in-site 참고용).
- **근거 + 출처등급**:
  - 환자 단위 = 누수 방지 [ML 기본].
  - 튜닝을 타깃 B에서 하면 타깃 누수 → **튜닝은 A-val로만**, B는 평가 때만 [확인됨: review HOLD 3].
  - A/B 유병률 다름(A 8.8% vs B 5.71%)이라 cross-site가 일반화의 핵심 시험대 [확인됨: EDA §8 / research/01·04].
- **고려한 대안**: 행 단위 random → 환자 걸침(누수). 양방향(A→B, B→A) → 단순화 위해 단방향. val 출처 미정(v1) → 타깃 누수 위험(HOLD 3).
- **미결/옵션**: A-train/A-val 비율·seed = 핸드오프 디테일.
- **검토 요청 항목**: **B가 학습·튜닝에 전혀 닿지 않는지**, A-val이 A-train 환자와 겹치지 않는지.

## 결정 6: baseline 입력 표현 — 윈도우 요약통계 (XGBoost)

- **결정**: baseline = **XGBoost**. 입력은 8시간 윈도우를 **요약통계**(마지막값·평균·최소·최대·기울기 등)로 압축한 한 줄, NaN은 그대로. 윈도우 경계(8h)는 **GRU와 동일**.
- **근거 + 출처등급**:
  - "윈도우 요약통계 + 부스팅"은 챌린지 표준 [확인됨: review가 리뷰가 아닌 상위팀 1차 논문 Separatrix 3위·TASP 4위·Sepyd 2위로 확인].
  - **정정**: 우승팀(Morrill)은 path signature + 도메인 파생피처(ShockIndex·BUN/CR·PartialSOFA)를 썼고, **요약통계도 함께** 사용함 [확인됨: review, CinC2019-014 + repo]. 우리 baseline은 그 *가벼운 요약통계* 버전.
  - XGBoost 선택 [우리 결정 — 결측 내장 처리].
- **고려한 대안**: (가) 시각별 행 → 추세 못 봄. (나) 평탄화 128열 → 차원 폭발·순서 상실·빈칸 8배.
- **미결/옵션**: 없음. **ShockIndex 등 도메인 파생피처는 baseline에 직접 넣지 않고 H2 ablation 항목으로 분리**("가벼운 하한선" 원칙 보존, review 권고).
- **검토 요청 항목**: 요약통계 목록 중 **"slope(기울기)"가 상위팀에서 실제 쓰였는지** 1차 출처 재확인(미확인 항목, review 지적). 결측이 많은 검사의 요약통계가 의미 있는지(전부 NaN인 윈도우 처리).
- **⚠️ 벤치마크 함정**: AUC 0.98류 보고치는 누수 의심 → 목표 금지. 공식 utility(우승 ~0.36) 기준 유지.

## 결정 7: 결측 마스크 — 기본 OFF (옵트인 자리만)

- **결정**: 결측 마스크 채널 **기본 미사용**. H1은 생성 코드 + off 플래그 **자리만**. 켜는 검증(A·B)은 H3.
- **근거 + 출처등급**:
  - 측정밀도/결측이 정보를 담는다(informative missingness)는 **확립된 현상** — 검사 측정 유무가 예측력을 가짐 [확인됨: Sharafoddini 2019 등], 측정 빈도는 값의 비정상도와 연관 [확인됨: JMIR 2025].
  - **특히 cross-site에서 위험**: 측정 빈도·결측률 같은 관찰-과정 피처는 site 내 판별력은 올리나 **cross-site 이식성을 악화**시킴 [확인됨: medRxiv 2026 MIMIC-IV↔eICU]. → 우리 A→B 시나리오 정통.
  - 치료행동 누수 통로(Epic 외부검증 실패) [확인됨: research/04].
  - *단,* PhysioNet에서의 구체적 발현(발병 직전 측정밀도 상승)은 결정 4의 EDA로 확인 예정 [검증 필요 → 데이터로].
- **고려한 대안**: 기본 ON → 미검증 누수 채택. 미구현 → H3 검증 불가.
- **미결/옵션**: 채택 여부는 H3 검증 결과에 달림.
- **검토 요청 항목**: off가 진짜 기본인지, 마스크 생성이 미래 정보를 안 쓰는지.

## 결정 8: 캐싱 레이어 — raw(NaN 보존)를 한 번만, 변환은 런타임

- **결정**: 후보 피처를 **NaN 그대로** + 라벨 + 사이트로 환자별 raw 배열로 **한 번** 캐싱(EtCO2 포함 전체 후보 보존 — 추후 재포함 여지). 4만 파일 읽기는 이때 한 번뿐. **피처셋 선택·분할·ffill·마스크·정규화·윈도잉은 모두 런타임.** 런타임 순서: 피처셋 슬라이스 → 분할 → (GRU)ffill → (옵트인 시)마스크 → train평균 fill → train통계 정규화 → 윈도잉 / (XGB)요약통계.
- **근거 + 출처등급**: NaN 보존해야 XGBoost(그대로)·GRU(ffill) 두 경로를 한 캐시로 커버 [우리 결정]; 풀셋 캐싱 → 활력셋은 컬럼 슬라이스로 파생 [우리 결정]; 재학습 다회 비용 절감 핵심 레버, CPU로 충분(GPU 불필요) [우리 결정].
- **고려한 대안**: ffill·정규화까지 캐싱 → 모델별 분기·분할별 정규화 불가(누수/경직).
- **미결/옵션**: 캐시 포맷(npz/parquet) = 핸드오프 디테일.
