## 한 줄 요약

H1이 만든 시퀀스(GRU)·요약통계(트리)를 써서 **세 모델(XGBoost·LightGBM·GRU)을 두 피처셋(활력 9 / 활력+검사 18)으로** 학습한다. 학습·튜닝·평가는 모두 **A 안에서만**(A-train 학습, A-val로 PR-AUC+utility 평가·선택), **B는 끝까지 봉인**(H3에서만 펼침). 끝에서 사람이 "어느 피처셋·어느 baseline을 대표로"를 판단한다.

## H2 범위 / 범위 외

| 범위 (H2가 하는 것) | 범위 외 (H3+) |
|---|---|
| 6개 조합(모델3 × 피처셋2) 학습 | B 펼쳐 cross-site 최종 채점 (H3) |
| A-val 평가(PR-AUC + utility) | cross-site gap 분석 (H3) |
| utility 구현체(A-val에 사용) | utility 정밀 검증·우승점수 분해 TSV 대조 (H3) |

| MLflow 실험 기록 | 피처 엔지니어링 ablation 2차 바퀴 (이연) |
| 하이퍼파라미터 A-val 탐색 | 서빙·운영 (H4) |

## 결정 1: 학습 대상 — 세 모델, baseline vs 메인 구도

- **결정**: **XGBoost·LightGBM**(baseline, 트리, 요약통계 입력) + **GRU**(메인, m2m, 시퀀스 입력) 학습. 트리 둘 중 A-val 우수자를 **대표 baseline**으로 뽑아 GRU와 비교(결정 7).
- **근거 + 출처등급**:
  - baseline = "GRU가 넘어야 할 강한 기준선" [우리 결정]. 우승팀이 LightGBM이라 트리는 허수아비가 아닌 진짜 경쟁자 [확인됨: research/01, H1 결정 6].
  - 핵심 질문: "시간 흐름을 통째로 배우는 GRU가 요약통계 트리를 이기나, 특히 cross-site에서" [우리 결정].
- **고려한 대안**: GRU만 학습(비교 대상 없음 → 성능 해석 불가). 트리 하나만(우승 계열 누락).
- **미결/옵션**: Transformer는 "욕심" 항목으로 H2 범위 외(여유 시 별도).
- **검토 요청 항목**: 세 모델이 동일 분할·동일 평가로 공정 비교되는지(변인 통제).

## 결정 2: 기본 ablation 축 — 피처셋 (활력 9 vs 활력+검사 18)

- **결정**: 기본 ablation은 **피처셋 하나만** 바꾼다 — `vitals`(9) vs `vitals_labs`(18, 검사 9종 추가). 나머지(분할·결측·정규화·seed·평가)는 전부 고정. **하이퍼파라미터는 피처셋마다 재탐색하지 않고, 모델별로 한 번 탐색해 동결한 값을 두 피처셋에 공통 적용**(결정 6) — 그래야 검사값이 유일 변인. → 6개 조합(모델3 × 피처셋2).
- **근거 + 출처등급**:
  - 검사 90%+ 결측이라 "넣는 게 나은지 노이즈인지" 머리로 못 정함 → 측정 [확인됨: EDA / H1 결정 1].
  - 변인 통제: 검사값**만** 유일 변인이라야 "검사 효과"를 인과적으로 귀속 [우리 결정 — 실험 방법론].
- **고려한 대안**: 피처셋·피처엔지니어링·하이퍼파라미터를 동시에 변주 → 변인 다중화로 효과 귀속 불가.
- **미결/옵션**: **피처 엔지니어링 ablation(요약통계만 / +추세 / +도메인파생 / signature)은 2차 바퀴로 이연** — 기본 요약통계로 파이프라인 한 바퀴 검증 후 축으로 추가(부속 결정).
- **검토 요청 항목**: featureset 슬라이스(`config.featureset_indices`)가 두 피처셋을 정확히 분리하는지, EtCO2가 양쪽에서 배제되는지.

## 결정 3: 분할 — H1 3분할 재사용, B 봉인 (= H2/H3 경계)

- **결정**: H1 `split.py`의 **cross_site 3분할**(A-train / A-val / B봉인)을 그대로 사용. **H2는 A-train·A-val만** 접촉. **B는 H2에서 절대 미사용**(학습·튜닝·정규화통계·pos_weight·모델선택 어디에도). B를 펼치는 건 H3.
- **근거 + 출처등급**:
  - H1에서 누수 가드 검증 완료(환자단위·B봉인) [확인됨: H1-b 게이트 10/10].
  - **H2/H3 경계 = "B를 보느냐"** (지표가 아니라 데이터가 경계). 튜닝을 B로 하면 타깃 누수 [확인됨: H1 결정 5 / review HOLD 3].
- **고려한 대안**: unified 분할로 학습 → cross-site 일반화 측정 불가(참고용으로만 병행 가능).
- **미결/옵션**: 없음.
- **검토 요청 항목**: H2 학습/평가 코드 전 경로에서 B가 로드되지 않는지(정적·동적 확인).

## 결정 4: 평가 지표 — PR-AUC + utility, **둘 다 A-val부터**

- **결정**: 두 지표를 본다 — **PR-AUC**(불균형 표준, 임계값 불필요) + **공식 utility score**(PhysioNet 2019, 이진 예측 기반). **A-val에서 둘 다 계산**해 튜닝·선택에 사용. **정확도는 사용 금지**(양성 1.8%). utility 구현체는 H2에서 만들어 A-val부터 적용.
  - **utility 정의(공식 `evaluate_sepsis_score.py` 대조, 인라인)** [확인됨]: 보상 창 `dt_early=−12h ~ dt_late=+3h`, 최대 보상 `dt_optimal=−6h`. `max_u_tp=+1`, `min_u_fn=−2`, 기울기 `m1=+1/6`(TP 상승)·`m2=−1/9`·`m3=−2/9`. 정규화 = `(observed − inaction) / (best − inaction)` → 전부음성(=inaction)은 **0.0**, 완벽예측은 **1.0**. **per-patient 시계열 지표**(점 단위 아님).
  - **임계값(τ) 정책** ⭐ [우리 결정]: 모델은 확률을 내므로 utility 계산에 이진화 임계값 τ가 필요. **τ는 A-val에서 utility 최대화로 선정·동결**. **H3에서 B 채점 시 동결된 τ 그대로 사용(B에서 재탐색 금지 — 타깃 누수 방지)**. τ는 모델·featureset별로 저장(아티팩트).
- **근거 + 출처등급**:
  - 양성 1.8%라 정확도 무의미(전부 음성=98.2%) [확인됨: EDA].
  - PR-AUC = 불균형 표준 [ML 일반].
  - utility = 대회 공식, 타이밍 반영(조기 보상) [확인됨: research/03 + 공식 코드 1차 대조].
  - **선택 기준을 최종 목표(utility)와 정렬**하려면 A-val에서 utility를 봐야 함 [우리 결정].
- **고려한 대안**: H2는 PR-AUC만, utility는 H3에서 → 선택 기준과 최종 목표 불일치. τ를 B에서 선정 → 타깃 누수. 둘 다 기각.
- **미결/옵션**: utility **정밀 검증**(우승점수 ~0.36 분해 TSV 대조, research/03 12시점 표 대조)은 H3에 둠 — *쓰는 것*(A-val)과 *정밀 검증*은 분리.
- **검토 요청 항목**:
  - 핸드오프 자립성: 위 수치(dt·m1/m2/m3·정규화식)를 핸드오프에 **인라인**(research/03 참조만 금지).
  - sanity: 전부음성→0.0, 완벽예측→1.0, **research/03의 검증된 12시점 표와 시점별 점수 일치**.

## 결정 5: 학습 설정 — 모델별

- **결정**:
  - **트리(XGB·LGBM)**: 입력 = 매시점 요약통계(H1 `features.py`). NaN 그대로. 불균형 = `scale_pos_weight`에 H1 산출 pos_weight. 조기종료 = A-val 기준.
  - **GRU(m2m)**: 입력 = 시퀀스(H1 `sequence.py`). per-timestep BCE + pos_weight, **loss 마스킹으로 패딩 제외**. **단방향·우측패딩**(H1 전제). 조기종료 = A-val loss.
- **근거 + 출처등급**: 전부 H1에서 확정·검증된 설정 [확인됨: H1 결정 2·4·6, m2m 재스모크 4/4].
- **고려한 대안**: 없음(H1 결정 계승).
- **미결/옵션**: **긴 시퀀스(최대 331h) GRU 학습** — 메모리/BPTT 처리. 재스모크는 소규모였으므로 풀 규모에서 truncated BPTT·시퀀스 길이 상한 등이 필요한지 H2에서 확인.
- **검토 요청 항목**: loss 마스킹이 풀 학습에서도 패딩을 학습·평가서 제외하는지, pos_weight가 A-train(또는 통합 train)에서 산출되는지.

## 결정 6: 하이퍼파라미터 — 모델별 1회 탐색 → 동결, 공정 예산

- **결정**:
  - **탐색 단위**: HP는 **모델별로 `vitals_labs`(18)에서 한 번만 A-val로 탐색** → **동결** → 두 피처셋(vitals/vitals_labs)에 **동일 적용**. 피처셋마다 재탐색 안 함(featureset 유일 변인 — 결정 2).
  - **탐색 예산(공정)**: 모델당 **동일 trial 수 N**(예: N=20)·**동일 탐색 방법**(예: random search). 한 모델 과탐색 금지를 trial 수로 강제.
  - **튜닝 objective 단일화**: HP/모델 **선택 기준 = A-val utility**(결정 4·7과 정렬). *GRU 조기종료만 A-val loss 사용*(학습 종료 신호일 뿐 선택 기준 아님).
  - **탐색 범위**(아래, [우리 결정] — default 기반 관행):
    - 트리 공통: `lr ∈ [0.01, 0.3]`, `subsample/colsample ∈ [0.6, 1.0]`, `L2 ∈ [0, 10]`, `n_estimators`는 조기종료로 상한, `scale_pos_weight = H1 산출값(±소수 배수 후보)`.
    - XGBoost: `max_depth ∈ {3,5,7}`. LightGBM: `num_leaves ∈ {15,31,63}`.
    - GRU: `hidden ∈ {32,64,128}`, `layers ∈ {1,2}`, `lr ∈ [1e-4,1e-2]`, `dropout ∈ [0,0.3]`.
- **근거 + 출처등급**:
  - 최적값은 데이터 의존이라 탐색이 정석 [ML 일반].
  - **라이브러리 default는 [확인됨: 공식 문서]** — XGBoost `eta=0.3, max_depth=6, min_child_weight=1, subsample=1` / LightGBM `num_leaves=31, learning_rate=0.1`. **단 공식 문서엔 numeric 탐색범위가 없음** → 위 탐색범위는 **[우리 결정]**(default 기반 관행), `[확인됨]` 아님.
  - 데이터 제약(불균형·155만 행) [확인됨: EDA]. 공정 비교라 과튜닝 금지 [우리 결정].
- **고려한 대안**: 피처셋별 재탐색(변인 둘 → 결정 2 위반). 구체값 하드코딩(데이터 미반영). trial 수 미지정(공정 검증 불가).
- **미결/옵션**: N 구체값·search 방법 세부는 핸드오프에서 확정(단 모델 간 동일).
- **검토 요청 항목**: 모델 간 trial 수·search space가 실제로 동일한지(공정 assert), 선택이 utility 단일 기준인지.

## 결정 7: 대표 baseline 선정 — A-val 기준, B 미사용

- **근거 + 출처등급**: B로 고르면 타깃 누수 [확인됨: H1 결정 6 / review]. utility 우선 = 최종목표 정렬 [우리 결정].
- **고려한 대안**: B 점수로 선정(누수). 둘 다 GRU와 3자 비교(메시지 산만).
- **미결/옵션**: 없음.
- **검토 요청 항목**: 선정 로직이 A-val만 참조하는지.

## 결정 8: 재현성·실험 기록 — seed 고정 + MLflow

- **결정**: 모든 무작위(분할·초기화·셔플) **seed 고정**. **6개 조합 전부 동일 seed**(무작위 변인 통제). **MLflow에** 조합별 설정(모델·피처셋·하이퍼파라미터·seed)·결과(A-val PR-AUC+utility)·모델 아티팩트 기록.
- **근거 + 출처등급**:
  - seed 고정 = 재현성 + ablation 변인 통제(무작위 차이 제거) [ML 일반].
  - MLflow = 6+ 조합 추적·비교표 산출, H3에서 모델 아티팩트 재사용 [우리 결정 / pdm-mlops 자산].
- **고려한 대안**: seed 미고정(운이 변인에 섞임). 수기 기록(추적 불가).
- **미결/옵션**: MLflow 스토어 형식(file/db)은 핸드오프 디테일.
- **검토 요청 항목**: 6개 조합이 동일 seed·동일 분할을 쓰는지, 모델 아티팩트가 H3에서 로드 가능한 형식인지.

## 부속 결정

- **피처 엔지니어링 ablation 2차 바퀴(이연)**: 기본 요약통계로 6조합 한 바퀴 검증 후, 트리 입력 표현을 축으로 추가(요약통계만 / +추세 / +도메인파생 ShockIndex·BUN-CR·PartialSOFA / signature). "요약통계→signature→GRU 시간정보 보존 스펙트럼"을 측정. **H2 1차 바퀴엔 미포함**(변인 단일화) [우리 결정].
- **사람 체크포인트(H2 종료 시)** ⏸: 6조합 A-val 결과표를 보고 **(1) 어느 피처셋을 메인으로**(활력만 vs +검사), **(2) 대표 baseline**(XGB vs LGBM)을 사람이 판단. 자동 진행 아님.

## H2 PASS 기준 (핸드오프에 박을 게이트)

5. **성능 하한 (크리스프)**: 6조합 전부 A-val `PR-AUC ≥ 0.05`(랜덤 0.018의 ~2.7배 — 배선이 실제 학습; 수치는 핸드오프에서 확정).
